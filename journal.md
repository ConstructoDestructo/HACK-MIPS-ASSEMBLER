# Dev Journal — Hack → MIPS Assembler

> This is your learning record. After each phase or major milestone, write a short entry.
> Don't aim for polish — aim for honesty. Future-you (and interviewers) will thank present-you.

---

## How to Use This Journal

Each entry follows the same simple structure:

- **What I built** — a one or two sentence description of what got done
- **What tripped me up** — the bugs, misconceptions, or walls you hit
- **How I solved it** — the "aha moment" or the grind that got you through
- **What I'd do differently** — hindsight is free, use it
- **Key concept learned** — one concept you could now explain to someone else

Think of it like a ship's log. You don't write it for the smooth sailing — you write it for the storms, so the next captain (future you) knows where the rocks are.
